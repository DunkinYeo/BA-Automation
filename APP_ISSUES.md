## 이슈 목록

### APP-01 | 잘못된 시리얼 입력 시 에러 메시지 내용
- **상태**: ⚠️ 버그 의심
- **재현 방법**: 로그인 화면에서 존재하지 않는 시리얼 번호(예: 000000) 입력 → 연결하기 클릭
- **실제 팝업 텍스트**:
  - 제목: **알 수 없는 오류 (902)**
  - 본문: **알 수 없는 오류가 발생했습니다.**
  - 버튼: 확인
- **예상 동작**: TC 기준으로는 "검사 정보 없음" (ERR-03) 또는 "디바이스를 찾을 수 없음" (ERR-08/09) 등 사용자가 이해할 수 있는 메시지 표시 예상
- **확인 요청**: 서버 응답 코드 902의 의미 및 사용자에게 표시되어야 할 정확한 에러 메시지 텍스트 확인

---

### APP-02 | GPS 꺼짐 에러 팝업 텍스트
- **상태**: 🔍 확인 필요
- **재현 방법**: 스마트폰 위치(GPS) 기능 OFF → 시리얼 입력 후 연결하기 클릭
- **현상**: 에러 팝업이 표시되나 팝업 내 정확한 텍스트를 자동화 도구로 감지하지 못함
- **확인 요청**: 해당 에러 팝업의 정확한 한국어/영어 텍스트 확인 (제목, 본문, 버튼 텍스트 모두)

- **현상**: APP-02와 동일하게 팝업 텍스트 감지 실패
- **확인 요청**: 해당 에러 팝업의 정확한 한국어/영어 텍스트 확인 (제목, 본문, 버튼 텍스트 모두)

### APP-05 | 시리얼 미입력 시 "연결하기" 버튼 비활성화 상태 미반영
- **상태**: ⚠️ 버그 의심
- **재현 방법**: 로그인 화면에서 시리얼 입력 없이 "연결하기" 버튼 확인
- **현상**: 버튼이 시각적으로 비활성화(흐리게) 표시되는 것으로 보이나, Appium을 통한 `enabled` / `clickable` 속성 조회 시 둘 다 `true`로 반환됨
- **예상 동작**: 시리얼 미입력 상태에서 버튼 비활성화 시 `enabled` 또는 `clickable` 속성 중 하나가 `false`로 반환되어야 함 (TC-LOGIN-002 기준)
- **확인 요청**: React Native 버튼 비활성화 구현 방식 확인 — 접근성 속성(`aria-disabled` 등) 또는 별도 disabled 처리 여부
